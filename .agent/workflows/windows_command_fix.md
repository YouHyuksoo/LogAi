## 윈도우 터미널 명령어 실행 가이드

Windows OS에서 명령어가 실행되지 않거나, 실행 결과(stdout)를 에이전트가 읽지 못하는 경우 아래의 규칙을 반드시 준수한다.

### 1. 명령어 래핑 (cmd /c & ::)

모든 터미널 명령어는 `cmd /c`로 감싸고, 명령어 끝에 `& ::`를 붙여서 뒤에 따라오는 잘못된 특수 문자나 따옴표를 흡수하도록 한다.

- **잘못된 예**: `dir`, `python main.py`
- **올바른 예**: `cmd /c dir & ::`, `cmd /c python main.py & ::`

### 2. 자동 실행 정책 (Turbo/Auto)

워크플로우 파일 내에 `// turbo` 또는 `// turbo-all` 주석이 있는 경우 `SafeToAutoRun`을 `true`로 설정하여 승인 대기 시간을 최소화한다.

### 3. 터미널 블라인드니스 해결

쉘 시작 파일(.bashrc, profile 등)에서 불필요한 출력을 제거하여 에이전트가 순수한 결과값만 캡처할 수 있도록 한다.
